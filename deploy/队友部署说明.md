# 队友部署说明（开箱即用 / 完整环境快照）

本包是**完整环境快照**：恢复后知识库（97 个文档）、工作流、模型密钥、Tavily 密钥全部就位，无需自己配置。

## 前置要求

- 已安装 **Docker Desktop**（Windows/Mac）并已启动
- 机器能访问 `api.siliconflow.cn`（模型）和 `api.tavily.com`（联网搜索）
- 磁盘剩余空间 ≥ 5 GB

## 部署步骤（3 步）

### 1. 启动 Dify 容器（会自动 clone 官方 Dify）

```bash
# Mac/Linux
bash deploy/startup.sh

# Windows PowerShell
.\deploy\startup.ps1
```

等所有容器起来（首次拉镜像较慢，约 5-10 分钟）。

### 2. 恢复环境快照

```bash
# Mac/Linux
bash deploy/restore-snapshot.sh snapshot/

# Windows PowerShell
.\deploy\restore-snapshot.ps1 -Snapshot .\snapshot
```

脚本会自动：就位 `.env`（含 SECRET_KEY）→ 恢复 PostgreSQL → 恢复 Weaviate 向量 → 重启服务。

### 3. 访问

等约 30 秒服务就绪，浏览器打开：

```
http://localhost
```

用 `snapshot/env.backup` 里 `INIT_PASSWORD` 对应的账号登录（或看下方说明）。

## 常见问题

**Q: 模型报错 / 回答为空？**
A: 检查 SECRET_KEY 是否与源一致（恢复脚本已自动处理）。若手动改过 `.env`，模型密钥会解不开——重新用 `snapshot/env.backup` 覆盖 `dify/docker/.env` 再重启。

**Q: 联网分支 401？**
A: Tavily 密钥已在工作流节点中。若失效，去 tavily.com 换新 key，在工作流 Tavily 节点 Body 替换。

**Q: 502 Bad Gateway？**
A: 容器还没完全就绪，等 30 秒刷新；或 `cd dify/docker && docker compose restart nginx`。
