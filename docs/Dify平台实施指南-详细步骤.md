## 三、步骤 2：配置 DeepSeek 模型供应商

> ⚠️ **关键**：模型名必须填 `deepseek-v4-flash`，不能用 `deepseek-chat`（2026-07-24 停用）

### 操作步骤

**2.1 进入设置页面**

1. 点击左下角**你的头像** → **「设置」**
   - 位置：页面最左下角，头像图标旁边

**2.2 进入模型供应商页面**

2. 在左侧设置菜单中，点击 **「模型供应商」**
   - 图标：一个芯片/大脑样式的图标
   - 文字：模型供应商

**2.3 搜索 DeepSeek**

3. 在模型供应商列表中，找到 **「DeepSeek」**
   - 如果列表中没有，使用顶部的搜索框输入 `DeepSeek`
   - 如果还是没有，选择 **「OpenAI-API-compatible」** 代替

**2.4 添加模型**

4. 点击 DeepSeek 卡片上的 **「添加模型」** 或 **「设置」** 按钮

5. 填写以下配置：

| 配置项 | 填写值 | 说明 |
|--------|--------|------|
| API Key | 粘贴 `.env` 中的 `DEEPSEEK_API_KEY` | 以 `sk-` 开头 |
| API Base URL | `https://api.deepseek.com` | 不需要尾部 `/v1` |
| 模型类型 | 选择 **「文本生成」** | — |
| 模型名称 | `deepseek-v4-flash` | ⚠️ 不要选 deepseek-chat |
| 模型模式 | 选择 **「聊天（Chat）」** | — |
| 上下文长度 | `1000000` | 1M token |
| 最大 Token | `8192` | — |

6. 点击 **「保存」**

**2.5 验证**

7. 在模型列表中，找到刚添加的 `deepseek-v4-flash`
8. 点击旁边的 **「测试」** 按钮
9. 输入 `你好`，点击发送
10. 应返回正常回复 → **✅ 通过**

### 如果找不到 DeepSeek 供应商

使用「OpenAI-API-compatible」替代：

```
1. 模型供应商 → 搜索「OpenAI-API-compatible」
2. 点击「添加模型」
3. 填写：
   - API Key：粘贴 DeepSeek Key
   - API Endpoint：https://api.deepseek.com
   - 模型名称：deepseek-v4-flash
   - 模型类型：文本生成（聊天）
```

## 四、步骤 3：启用 BGE Embedding

### 操作步骤

1. 保持在 **「设置」→「模型供应商」** 页面
2. 在列表中找 **「BGE」** 或 **「Sentence Transformers」**
3. 点击进入
4. 找到 `bge-large-zh-v1.5`
5. 如果状态是「未安装」，点击 **「安装」**
6. 等待安装完成（约 30 秒）
7. 状态变为「可用」→ **✅ 通过**

> 如果找不到 BGE，备选方案：
> - 使用 Dify 内置的 `text-embedding-3-small`（OpenAI 兼容，需额外付费）
> - 或使用本地模型 `m3e-base`

### 5.5 分段设置（关键！）

|--------|----------|-------------|

| 分段规则 | **按章节标题** | 下拉框选择 |

| 分段重叠 | 输入 **`50`** | 输入框中填入 |

| 配置项 | 值 | 界面位置 |

| 检索模式 | **「混合检索」** | 进入知识库 → 检索设置 |
| Top K | **3** | 输入框 |
| Score 阈值 | **0.5** | 输入框 |
| Rerank | **开启** | 开关按钮 |

1. 在知识库页面，点击 **「召回测试」** 标签
2. 输入测试查询：`主轴异响 报警代码 SP-2003`

4. 应返回相关片段，且 Score ≥ 0.5 → **✅ 通过**

## 六、步骤 5：配置 Tavily 搜索（用 HTTP 请求节点）

> ⚠️ Dify 1.16.0 已移除旧版「自定义工具」，改用 Workflow 的 **「HTTP 请求」** 节点替代。
> 不需要 marketplace，不需要插件——直接在画布上配置。

### 5.1 在 Workflow 中添加 HTTP 请求节点

1. 进入 Workflow 应用编辑画布
2. 在左侧工具栏 → 找到 **「HTTP 请求」** 节点
3. 拖拽到画布上，放在 if-else 的 **false 分支** 上

### 5.2 配置节点

双击 HTTP 请求节点，在右侧配置面板填写：

| 配置项 | 填写值 |
|--------|--------|
| 请求方法 | `POST` |
| URL | `https://api.tavily.com/search` |
| Headers | 添加一行：`Content-Type` = `application/json` |
| Body 类型 | 选择 **JSON** |

**Body 内容**（粘贴以下 JSON）：

```json
{
  "api_key": "你的Tavily Key",
  "query": "{{#sys.query#}}",
  "max_results": 5,
  "search_depth": "basic",
  "include_answer": true
}
```

> ⚠️ `"api_key"` 处填入 `.env` 里的 `TAVILY_API_KEY` 实际值。

**超时设置**：

| 配置项 | 值 |
|--------|-----|
| 连接超时 | 10 |
| 读取超时 | 30 |
| 写入超时 | 10 |

### 5.3 连接节点

- if-else `false` 输出 → HTTP 请求节点
- HTTP 请求节点 → LLM 节点

### 5.4 在 LLM 节点中引用搜索结果

LLM 节点的 User Prompt 中，引用 HTTP 请求的输出：

```
--- 联网搜索结果（可能为空）---
{{#tavily_search.body#}}
```

### 5.5 测试

1. 输入一个知识库不覆盖的查询（如「量子计算机维修」）
2. 应走 false 分支 → HTTP 请求 → 返回 Tavily 搜索结果

```
# 回答规则

## 1. 内容来源约束
- 严格基于检索到的资料回答，不编造未在资料中出现的信息
- 如果知识库结果非空，优先使用知识库内容；知识库为空时使用联网搜索结果

## 2. 答案结构（必须遵循）
答案必须按以下结构输出：

**故障可能原因：**
1. [原因1]（概率 XX%）
2. [原因2]（概率 XX%）

**排查步骤：**
Step 1: [可执行的排查动作]
Step 2: [可执行的排查动作]
⚠️ Step 3: [涉及安全的操作必须加警告标识]

**引用来源：**
- 📄 [文档名] 第X页（知识库命中时）
- 🔗 [URL]（联网搜索时）

## 3. 多轮对话上下文处理
- 当用户追问时，结合历史对话保持设备型号、故障类型等上下文一致
- 如果用户切换到完全新话题，直接回答新问题

## 4. 兜底话术（无法回答时）
⚠️ 现有资料无法完全解答该问题。建议：1.联系设备厂商技术支持 2.提供更多故障细节

## 5. 安全警告规则
- 涉及高压电操作、机械拆卸、压力系统的步骤，必须加 ⚠️ 警告
```

7. 关闭配置面板

   - 引用来源中有 📄 文档名
5. 输入另一个查询：`西门子 840D sl 驱动过流怎么处理`
6. 应返回含 🔗 URL 的结果 → **✅ 通过**

## 八、步骤 7：获取 API Key 供测试脚本使用

### 8.1 获取 App API Key

1. 在 Workflow 应用页面左侧菜单 → 点击 **「API 访问」**
2. 点击 **「创建 API Key」**
3. 复制 Key（格式：`app-xxxxxxxx`）
4. 记录命令：
   ```bash
   python src/tests/test_scenarios.py --api_key app-xxxxxxxx --base_url http://localhost/v1
   ```

### 8.2 获取 Knowledge API Key 和 Dataset ID

1. 回到知识库页面
2. 点击知识库 **「制造业设备维修手册库」**
3. 在左侧菜单 → 点击 **「API 访问」**
4. 点击 **「创建 API Key」** → 复制（格式：`dataset-xxxxxxxx`）
5. 查看浏览器地址栏，复制 Dataset ID（URL 中的 UUID）
   - 例如：`http://localhost/datasets/abc123-def456-ghi789` → ID 是 `abc123-def456-ghi789`
6. 记录命令：
   ```bash
   python src/knowledge/verify_retrieval.py --dataset_id 你的ID --api_key dataset-xxxxxxxx
   ```

## 常见问题速查

| 问题 | 解决方法 |
|------|---------|
| 知识库召回测试无结果 | 降低 Score 阈值（0.5 → 0.3） |
