## 步骤 1：添加 DeepSeek 模型供应商

### 操作指引

1. 浏览器访问 `http://localhost`，登录 Dify 控制台
2. 左下角点击 **「设置」**（齿轮图标）
3. 选择 **「模型供应商」**
4. 在供应商列表中找到 **「SiliconFlow」**（如未显示，选择 **「OpenAI-API-compatible」**）
5. 点击 **「添加模型」**，填写以下配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 供应商名称 | SiliconFlow | 或自定义 |
| API Key | （从 `.env` 文件复制 `DEEPSEEK_API_KEY` 的值） | 注册地址 https://siliconflow.cn |
| API Base URL | `https://api.siliconflow.cn/v1` | SiliconFlow OpenAI 兼容端点 |
| 模型名称 | `deepseek-ai/DeepSeek-V4-Pro` | ⚠️ 不要用已停用的 `deepseek-chat` |
| 模型类型 | 文本生成（Chat） | |
| 模型模式 | 聊天（Chat） | |
| 上下文长度 | 1000000 | V4 标配 1M |
| 最大 Token 上限 | 8192 | 输出上限 |

6. 点击 **「保存」**

### 验证方式

- 在模型列表中看到 `deepseek-ai/DeepSeek-V4-Pro` 且状态为「可用」
- 点击模型旁的 **「测试」** 按钮，输入「你好」，应返回正常回复
- 命令行验证（可选）：

```bash
# 从 .env 读取 Key 并测试
DEEPSEEK_API_KEY=$(grep "^DEEPSEEK_API_KEY=" dify/docker/.env | cut -d'=' -f2-)
curl -s https://api.siliconflow.cn/v1/chat/completions \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"deepseek-ai/DeepSeek-V4-Pro","messages":[{"role":"user","content":"ping"}],"max_tokens":10}'
```

### ⚠️ 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 模型列表不显示 V4 | Dify 版本过旧 | 升级 Dify 或用「OpenAI-API-compatible」方式接入 |
| HTTP 401 | API Key 错误 | 重新从 https://siliconflow.cn 获取 |
| HTTP 404 | 模型名错误 | 确认是 `deepseek-ai/DeepSeek-V4-Pro`，不是已停用的 `deepseek-chat` |

## 步骤 2：创建知识库并上传手册

### 操作指引

1. 在 Dify 控制台顶部导航栏点击 **「知识库」**
2. 点击 **「创建知识库」**
3. 上传手册 PDF 文件（FANUC 0i-MF 维修手册、Siemens 840D sl 操作手册等公开版）
4. **分段设置**（关键配置）：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 分段方式 | 自动 | Dify 自动识别章节 |
| 分段长度 | 500 tokens | PRD 4.3 节规定 |
| 分段重叠 | 50 tokens | 避免边界信息丢失 |
| 分段规则 | 按章节标题（H1/H2） | 设备手册结构清晰 |

5. **索引方式**：选择「高质量索引」（使用 Embedding 模型）
6. **Embedding 模型**：选择 `bge-large-zh-v1.5`
   - 如未安装，在「设置 → 模型供应商」中找到 BGE 并启用
7. **检索设置**：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 检索模式 | 混合检索（推荐） | 语义 + 关键词 |
| Top K | 3 | PRD 3.2 节规定 |
| Score 阈值 | 0.5 | PRD 3.2 节规定 |
| Rerank | 开启 | 使用 Dify 内置 reranker |

8. 点击 **「保存并处理」**，等待向量化完成

### 验证方式

- 知识库列表中显示状态为「可用」
- 点击知识库 → **「召回测试」**
- 输入测试查询：「主轴异响 报警代码」
- 应返回相关手册片段，且 Score ≥ 0.5

### ⚠️ 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| PDF 解析失败 | 扫描件无文字层 | 用 OCR 工具预处理（如 PaddleOCR） |
| Embedding 模型不可用 | 未安装 BGE | 在模型供应商中启用 BGE |
| 检索结果不相关 | 分段过大/过小 | 调整分段长度（建议 300-800 tokens） |

## 步骤 3：配置 Tavily 自定义工具

### 操作指引

> **⚠️ 决策说明**：经确认，Tavily 通过 Dify 原生自定义工具接入（方式 A）。
> 站点定向搜索功能（site: 语法）降级，MVP 阶段仅支持通用搜索。
> 用户可在查询时手动添加 `site:fanuc.com` 等语法实现定向。

1. 在 Dify 控制台点击 **「工具」** → **「自定义」** → **「创建自定义工具」**
2. 填写基本信息：

| 配置项 | 值 |
|--------|-----|
| 工具名称 | Tavily 搜索 |
| 工具描述 | 联网搜索制造业设备维修资料 |

3. **认证方式**：选择「API Key」
4. **Schema**（粘贴以下 OpenAPI YAML）：

```yaml
openapi: 3.0.0
info:
  title: Tavily Search
  version: 1.0.0
  description: Web search API for manufacturing equipment repair info
servers:
  - url: https://api.tavily.com
paths:
  /search:
    post:
      operationId: tavilySearch
      summary: Search the web for equipment repair information
      description: Returns relevant web pages with snippets and URLs
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                query:
                  type: string
                  description: Search query, e.g. "CNC spindle noise alarm SP-2003"
                max_results:
                  type: integer
                  description: Number of results to return
                  default: 5
                search_depth:
                  type: string
                  enum: [basic, advanced]
                  default: basic
                  description: basic for speed, advanced for thoroughness
                include_answer:
                  type: boolean
                  default: true
                  description: Include a short AI-generated answer
              required:
                - query
      responses:
        '200':
          description: Search results
          content:
            application/json:
              schema:
                type: object
                properties:
                  answer:
                    type: string
                    description: AI-generated summary answer
                  results:
                    type: array
                    items:
                      type: object
                      properties:
                        title:
                          type: string
                        url:
                          type: string
                        content:
                          type: string
                          description: Snippet of the page content
```

5. 在 **「认证配置」** 中填入 Tavily API Key（从 `.env` 文件复制 `TAVILY_API_KEY` 的值）
6. 点击 **「保存」**
7. 点击 **「测试」**，输入查询「CNC machine repair」，应返回搜索结果

### 验证方式

- 工具列表中显示「Tavily 搜索」且状态为「可用」
- 测试调用返回 JSON 格式的搜索结果（包含 title、url、content 字段）
- 命令行验证（可选）：

```bash
TAVILY_API_KEY=$(grep "^TAVILY_API_KEY=" dify/docker/.env | cut -d'=' -f2-)
curl -s https://api.tavily.com/search \
  -H "Content-Type: application/json" \
  -d "{\"api_key\":\"$TAVILY_API_KEY\",\"query\":\"CNC repair\",\"max_results\":1}"
```

### ⚠️ 功能降级说明

| 功能 | PRD 原计划 | MVP 实际 | 后续迭代 |
|------|-----------|---------|---------|
| 通用搜索 | Tavily API | ✅ Tavily API | — |
| 专业站点定向 | site: 语法封装 | ❌ 降级（用户手动加 site:） | Phase 3 评估 FastAPI 中间层 |

## 步骤 4：创建 Workflow 应用

### 操作指引

1. 在 Dify 控制台点击 **「创建空白应用」**
2. 选择 **「工作流」（Workflow）** 类型
3. 填写应用名称：「制造业设备维修智能问答」
4. 进入工作流编辑器，按 PRD 3.2 节配置 6 个节点：

#### 节点 1：Start

| 配置项 | 值 |
|--------|-----|
| 输入变量 | `user_query`（string，必填） |
| 描述 | 用户输入的故障描述 |

#### 节点 2：Knowledge Retrieval（知识库检索）

| 配置项 | 值 |
|--------|-----|
| 知识库 | 选择步骤 2 创建的知识库 |
| 查询变量 | `{{#start.user_query#}}` |
| Top K | 3 |
| Score 阈值 | 0.5 |
| Rerank | 开启 |

#### 节点 3：Condition（条件判断）

| 配置项 | 值 |
|--------|-----|
| 条件 | `{{#knowledge_retrieval.result[0].score#}} >= 0.5` |
| 分支 A（命中） | 连接到 LLM 节点 1 |
| 分支 B（未命中） | 连接到 Tool 节点 |

#### 节点 4：Tool（搜索工具，仅分支 B）

| 配置项 | 值 |
|--------|-----|
| 工具 | Tavily 搜索（步骤 3 创建） |
| 输入 query | `{{#start.user_query#}}` |
| max_results | 5 |

#### 节点 5：LLM（综合生成，两个分支各一个）

| 配置项 | 值 |
|--------|-----|
| 模型 | `deepseek-ai/DeepSeek-V4-Pro` |
| 思考模式 | 开启 |
| reasoning_effort | high |
| 温度 | 0.3 |

**System Prompt**（粘贴以下内容）：

```
你是一位制造业设备维修智能助手，专注于 CNC 数控机床、PLC、工业机器人等设备的故障诊断与维修指导。

【回答规则】
1. 严格基于检索到的资料回答，不编造未在资料中出现的信息
2. 答案必须按以下结构输出：
   - 故障可能原因（按概率从高到低排序，标注概率百分比）
   - 排查步骤（分步骤编号，每步可执行）
   - 引用来源（每个关键结论后标注）
3. 引用格式：
   - 私有手册：【来源：文档名 第X页】
   - 网页资料：【来源：URL】
4. 如果检索资料不足以回答，明确告知"现有资料无法完全解答该问题，建议联系设备厂商技术支持"
5. 涉及安全操作的步骤，必须加 ⚠️ 警告标识
```

**分支 A 的 User Prompt**：

```
用户问题：{{#start.user_query#}}

知识库检索结果：
{{#knowledge_retrieval.result#}}

请基于以上检索结果回答用户问题。
```

**分支 B 的 User Prompt**：

```
用户问题：{{#start.user_query#}}

联网搜索结果：
{{#tool.body#}}

请基于以上搜索结果回答用户问题。
```

#### 节点 6：End

| 配置项 | 值 |
|--------|-----|
| 输出变量 | `answer`（引用 LLM 节点的输出） |

5. 点击右上角 **「发布」**

### 验证方式

- 工作流编辑器中无红色错误标记
- 点击 **「运行」** 测试：
  - 输入「FANUC 主轴异响」，应走分支 A（知识库命中）
  - 输入「西门子驱动过流」，应走分支 B（联网兜底）
- 检查输出是否包含「故障可能原因」「排查步骤」「引用来源」三部分

## 步骤 5：测试演示场景

### 操作指引

按 PRD 5.1 节验收标准，测试 3 个典型场景：

#### 场景 1：知识库命中

| 输入 | 「FANUC 0i-MF 系统主轴异响，报警代码 SP-2003」 |
|------|----------------------------------------------|
| 预期路径 | 分支 A（知识库命中） |
| 验收点 | 返回排查步骤 + 手册引用（文档名 + 页码） |
| 通过条件 | 答案结构完整，引用来源可点击 |

#### 场景 2：联网兜底

| 输入 | 「西门子 840D sl 驱动模块过流故障怎么处理」 |
|------|------------------------------------------|
| 预期路径 | 分支 B（联网搜索） |
| 验收点 | 返回搜索结果 + URL 引用 |
| 通过条件 | 答案相关，URL 可访问 |

#### 场景 3：多轮追问

| 第一轮输入 | 「CNC 加工中心换刀故障」 |
|-----------|----------------------|
| 第二轮输入 | 「如果换刀臂卡在中间位置怎么办」 |
| 验收点 | 第二轮基于第一轮上下文回答 |
| 通过条件 | 答案连贯，不重复第一轮内容 |

### 验证方式

- 3 个场景全部通过 → Demo 验收成功 ✅
- 任一场景失败 → 按「常见问题」排查

### ⚠️ 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 知识库未命中 | 手册内容未覆盖该故障 | 扩充知识库或调整 Score 阈值 |
| 联网搜索无结果 | 查询过窄或专业性强 | 在查询中手动加 `site:fanuc.com` 等 |
| LLM 回答结构不符 | System Prompt 未生效 | 检查 LLM 节点的 System Prompt 配置 |
| 响应超时（>30s） | 网络慢或思考模式耗时过长 | 降低 reasoning_effort 或关闭思考模式 |
