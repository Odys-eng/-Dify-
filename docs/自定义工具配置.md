## 一、Tavily 工具 OpenAPI Schema 完整版

### 1.1 在 Dify 中配置步骤

1. Dify 控制台 → **「工具」** → **「自定义」** → **「创建自定义工具」**
2. 填写基本信息后，粘贴以下 Schema

### 1.2 完整 OpenAPI Schema

```yaml
openapi: 3.0.0
info:
  title: Tavily Search API
  version: 1.0.0
  description: |
    Web search API optimized for AI agents.
    Used for manufacturing equipment repair information retrieval.
    Returns structured results with titles, URLs, and content snippets.

servers:
  - url: https://api.tavily.com
    description: Tavily API production endpoint

paths:
  /search:
    post:
      operationId: tavilySearch
      summary: Search the web for equipment repair information
      description: |
        Performs a web search and returns relevant results.
        Use this tool when the knowledge base does not have sufficient information.
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                query:
                  type: string
                  description: |
                    Search query string.
                    Example: "CNC spindle noise alarm SP-2003"
                  example: "FANUC 0i-MF spindle alarm SP-2003"
                max_results:
                  type: integer
                  description: Maximum number of results to return
                  default: 5
                  minimum: 1
                  maximum: 10
                search_depth:
                  type: string
                  enum:
                    - basic
                    - advanced
                  default: basic
                  description: |
                    basic: faster, less detailed
                    advanced: slower, more thorough
                include_answer:
                  type: boolean
                  default: true
                  description: Include a short AI-generated summary answer
                include_raw_content:
                  type: boolean
                  default: false
                  description: Include raw page content (increases response size)
                include_domains:
                  type: array
                  items:
                    type: string
                  description: |
                    Only search these domains.
                    Example: ["fanuc.com", "siemens.com"]
                  default: []
                exclude_domains:
                  type: array
                  items:
                    type: string
                  description: Exclude these domains from search
                  default: []
              required:
                - query
      responses:
        '200':
          description: Successful search response
          content:
            application/json:
              schema:
                type: object
                properties:
                  answer:
                    type: string
                    description: AI-generated summary answer (if include_answer=true)
                  query:
                    type: string
                    description: The original query
                  response_time:
                    type: number
                    description: Response time in seconds
                  results:
                    type: array
                    items:
                      type: object
                      properties:
                        title:
                          type: string
                          description: Page title
                        url:
                          type: string
                          description: Page URL
                        content:
                          type: string
                          description: Snippet of page content
                        score:
                          type: number
                          description: Relevance score (0-1)
                        raw_content:
                          type: string
                          description: Full page content (if include_raw_content=true)
        '401':
          description: Invalid API key
        '429':
          description: Rate limit exceeded (free tier: 1000 requests/month)
        '500':
          description: Internal server error

components:
  securitySchemes:
    ApiKeyAuth:
      type: apiKey
      in: header
      name: Authorization
      description: Bearer token authentication

security:
  - ApiKeyAuth: []
```

### 1.3 认证配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 认证方式 | API Key | Tavily 使用 Bearer token |
| Header 名称 | `Authorization` | 标准 Bearer 认证 |
| Key 值 | `Bearer {TAVILY_API_KEY}` | 从 .env 文件复制 |
| 认证位置 | Dify 工具配置页 | 创建工具时填入 |

### 1.4 工具测试查询

配置完成后，用以下查询测试：

| 测试查询 | 预期结果 |
|---------|---------|
| `CNC machine tool repair` | 返回 5 条相关网页 |
| `FANUC spindle alarm SP-2003` | 返回 FANUC 官网或工控论坛结果 |
| `Siemens 840D drive overcurrent` | 返回 Siemens 官网或技术文章 |

## 二、站点定向搜索工具方案对比

### 2.1 背景

PRD 5.2 节设计了「站点定向搜索」功能，通过 `site:` 语法限定搜索范围到制造业专业站点。但实现方式有两种，各有优缺点。

### 2.2 方案对比

| 维度 | 方案 A：Dify 原生自定义工具 | 方案 B：FastAPI 中间层服务 |
|------|--------------------------|-------------------------|
| **实现方式** | 在 Dify 自定义工具中配置 Tavily，用户查询时手动加 `site:` | 部署独立 FastAPI 服务，封装 Tavily + 自动追加 `site:` |
| **部署复杂度** | ✅ 零额外部署 | ❌ 需额外 Docker 容器 |
| **站点定向** | ❌ 手动（用户需记得加 `site:fanuc.com`） | ✅ 自动（服务内部处理） |
| **API Key 管理** | ⚠️ 在 Dify 工具配置中填入 | ✅ 环境变量管理 |
| **可维护性** | ✅ 高（Dify UI 配置） | ⚠️ 中（需维护代码） |
| **扩展性** | ❌ 低（固定 Schema） | ✅ 高（可加预处理/后处理） |
| **Demo 适用性** | ✅ 最快跑通 | ❌ 增加部署负担 |
| **与 PRD 一致性** | ⚠️ 降级（站点定向变手动） | ✅ 完整实现 |

### 2.3 推荐方案

> **推荐方案 A**（已由用户确认）
>
> **理由**：
> 1. Demo 阶段稳定性优先，减少部署组件
> 2. 用户可在查询时手动加 `site:fanuc.com OR site:gongkong.com` 实现定向
> 3. 如后续发现通用搜索覆盖不足，再评估方案 B

### 2.4 方案 B 参考代码（仅供后续迭代参考）

如需升级到方案 B，以下是 FastAPI 中间层的核心代码：

```python
"""
tavily_site_search.py
FastAPI 中间层：封装 Tavily + 自动追加 site: 语法
部署：docker compose 中新增一个服务
"""

import os
import requests
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI(title="Tavily Site-Directed Search")

TAVILY_API_KEY = os.environ.get("TAVILY_API_KEY", "")
TAVILY_URL = "https://api.tavily.com/search"

# 制造业专业站点列表
MANUFACTURING_SITES = [
    "site:fanuc.com",          # FANUC 官网
    "site:siemens.com",         # Siemens 官网
    "site:gongkong.com",        # 中华工控网
    "site:csdn.net",            # CSDN 技术文章
    "site:zhihu.com"            # 知乎工业话题
]


class SearchRequest(BaseModel):
    query: str
    max_results: int = 5
    search_depth: str = "basic"


class SearchResult(BaseModel):
    title: str
    url: str
    content: str
    score: float


@app.post("/search")
def site_directed_search(req: SearchRequest):
    """
    站点定向搜索：自动追加 site: 语法
    """
    # 构造增强查询
    site_clause = " OR ".join(MANUFACTURING_SITES)
    enhanced_query = f"{req.query} ({site_clause})"

    # 调用 Tavily API
    payload = {
        "api_key": TAVILY_API_KEY,
        "query": enhanced_query,
        "max_results": req.max_results,
        "search_depth": req.search_depth,
        "include_answer": True
    }

    try:
        response = requests.post(TAVILY_URL, json=payload, timeout=30)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.RequestException as e:
        raise HTTPException(status_code=502, detail=f"Tavily API error: {str(e)}")


@app.get("/health")
def health_check():
    """健康检查端点"""
    return {"status": "healthy", "service": "tavily-site-search"}
```

## 三、工具使用注意事项

### 3.1 Tavily 免费额度管理

| 项目 | 限制 |
|------|------|
| 免费额度 | 1000 次/月 |
| 超限处理 | 返回 429 错误 |
| Demo 消耗预估 | 演示 + 测试约 50-100 次 |
| 建议保留额度 | ≥ 800 次/月（留足缓冲） |

### 3.2 查询优化建议

| 场景 | 查询技巧 | 示例 |
|------|---------|------|
| 精确故障代码 | 直接搜索代码 | `SP-2003 alarm FANUC` |
| 设备型号 + 症状 | 组合关键词 | `Siemens 840D sl drive overcurrent` |
| 站点定向 | 手动加 site: | `CNC repair site:gongkong.com` |
| 中英文混排 | 保留英文型号 | `FANUC 0i-MF 主轴异响 alarm` |

### 3.3 结果解析

Tavily 返回的 JSON 结构：

```json
{
  "answer": "AI 生成的摘要",
  "results": [
    {
      "title": "网页标题",
      "url": "https://...",
      "content": "网页内容片段",
      "score": 0.95
    }
  ]
}
```

LLM 节点的 prompt 会引用 `{{#tavily_search.text#}}`，Dify 会将整个 JSON 作为文本传入。LLM 需要从中提取 title、url、content 信息。
