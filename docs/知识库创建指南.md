# Dify 知识库创建操作指南

> **文件路径**：`docs/知识库创建指南.md`
> **用途**：基于 PRD 第 4 节「知识库构建 Pipeline」，指导在 Dify 控制台创建知识库
> **前置条件**：已完成部署（`deploy/部署检查清单.md` 步骤 1-3）

---

## 创建流程总览

| 步骤 | 操作 | 关键配置 | 预计耗时 |
|------|------|---------|---------|
| 1 | 进入知识库创建页面 | — | 1 分钟 |
| 2 | 上传手册 PDF | 3 份文档 | 2 分钟 |
| 3 | 配置切片策略 | chunk_size=500, overlap=50 | 3 分钟 |
| 4 | 配置 Embedding 模型 | bge-large-zh-v1.5 | 2 分钟 |
| 5 | 配置检索参数 | top_k=3, score_threshold=0.5 | 2 分钟 |
| 6 | 等待向量化完成 | — | 5-10 分钟 |

## 步骤 1：进入知识库创建页面

### 操作指引

1. 浏览器访问 `http://localhost`，登录 Dify 控制台
2. 在顶部导航栏点击 **「知识库」** 标签
3. 点击页面右上角的 **「创建知识库」** 按钮

### 界面元素说明

```

│                                                 │
│   知识库                                        │
│   ┌──────────────────────────────┐              │
│   │  + 创建知识库  │  ← 点击此按钮 │              │
│   └──────────────────────────────┘              │
│                                                 │

```

## 步骤 2：上传手册 PDF

### 操作指引

1. 在「选择数据源」页面，选择 **「导入已有文本」**
2. 拖拽或点击上传以下 3 份 PDF 文档：

| 文档 | 用途 | 对应演示场景 |
|------|------|------------|
| FANUC 0i-MF 维修手册（公开版）.pdf | CNC 主轴/报警代码 | 场景 1（知识库命中） |
| Siemens 840D sl 操作手册（公开版）.pdf | 驱动模块/伺服 | 场景 2（联网兜底对照） |
| CNC 加工中心故障案例集.pdf | 换刀故障案例 | 场景 3（多轮追问） |

3. 点击 **「下一步」**

### ⚠️ 注意事项

- 文档必须是**文字版 PDF**（可复制文字），扫描件需先 OCR 处理
- 单份文档建议 < 50MB，超过可能上传超时
- 文档名建议用英文或纯中文，避免特殊字符

## 步骤 3：配置切片策略

### 关键配置项

| 字段名 | 推荐值 | 理由（对照 PRD 4.3 节） |
|--------|--------|----------------------|
| 分段方式 | **自动** | Dify 自动识别 H1/H2 章节标题，设备手册结构清晰 |
| 分段长度（chunk_size） | **500** | PRD 4.3 节规定，兼顾语义完整性和检索精度 |
| 分段重叠（overlap） | **50** | PRD 4.3 节规定，避免章节边界信息丢失 |
| 分段规则 | **按章节标题** | 设备手册天然按章节组织（报警代码表、维护规程等） |

### 界面配置示意

```
┌─────────────────────────────────────────────────┐
│  分段设置                                        │
│                                                 │
│  分段方式：  ○ 通用    ● 自动                    │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │  分段长度：  [  500  ]  tokens          │    │
│  │  分段重叠：  [  50   ]  tokens          │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  分段规则：  ● 按章节标题                        │
│             ○ 按固定长度                        │
│             ○ 按段落                            │
│                                                 │
└─────────────────────────────────────────────────┘
```

### ⚠️ 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 章节识别失败 | PDF 无书签/目录结构 | 改用「通用」分段方式，手动设置分隔符 |
| 分段过短（< 100 tokens） | 章节内容太少 | 降低分段长度或合并相邻章节 |
| 分段过长（> 1000 tokens） | 章节内容太多 | 降低分段长度到 300 |

## 步骤 4：配置 Embedding 模型

### 关键配置项

| 字段名 | 推荐值 | 理由 |
|--------|--------|------|
| 索引方式 | **高质量索引** | 使用 Embedding 模型，检索精度高 |
| Embedding 模型 | **bge-large-zh-v1.5** | PRD 4.4 节规定，中文 SOTA |
| Embedding 模型供应商 | BGE（内置） | Dify 社区版自带 |

### 操作指引

1. 在「索引方式」选择 **「高质量索引」**
2. 在「Embedding 模型」下拉框中选择 `bge-large-zh-v1.5`
   - 如下拉框无此选项，需先在「设置 → 模型供应商」中启用 BGE
3. 确认模型维度显示为 **1024**（bge-large-zh 的标准维度）

### 启用 BGE 模型（如未显示）

1. 回到 Dify 主页，点击左下角 **「设置」**
2. 选择 **「模型供应商」**
3. 找到 **「BGE」** 或 **「Sentence Transform」** 类别
4. 启用 `bge-large-zh-v1.5` 模型

## 步骤 5：配置检索参数

### 关键配置项

| 字段名 | 推荐值 | 理由（对照 PRD 3.2 节） |
|--------|--------|----------------------|
| 检索模式 | **混合检索** | 语义 + 关键词，兼顾精准和召回 |
| Top K | **3** | PRD 3.2 节规定，返回前 3 个最相关片段 |
| Score 阈值 | **0.5** | PRD 3.2 节规定，低于此分走联网兜底 |
| Rerank | **开启** | 使用 Dify 内置 reranker 提升精度 |
| Rerank 模型 | 默认（或 bge-reranker-base） | 中文场景推荐 |

### 界面配置示意

```
┌─────────────────────────────────────────────────┐
│  检索设置                                        │
│                                                 │
│  检索模式：  ● 混合检索（推荐）                   │
│             ○ 语义检索                          │
│             ○ 全文检索                          │
│                                                 │
│  Top K：          [  3  ]                       │
│  Score 阈值：     [ 0.5 ]                       │
│                                                 │
│  Rerank：  ☑ 开启                               │
│  Rerank 模型：  [ bge-reranker-base ▼]          │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 步骤 6：等待向量化完成

### 操作指引

1. 确认所有配置后，点击 **「保存并处理」**
2. 页面跳转到知识库详情，显示文档处理进度
3. 等待所有文档状态变为 **「可用」**（绿色图标）

### 处理时间预估

| 文档页数 | 预计处理时间 |
|---------|------------|
| < 100 页 | 2-3 分钟 |
| 100-300 页 | 5-8 分钟 |
| 300-500 页 | 10-15 分钟 |
| > 500 页 | 15-30 分钟 |

## 验证：召回测试

### 操作指引

1. 在知识库详情页，点击 **「召回测试」** 标签
2. 输入测试查询，例如：「主轴异响 报警代码」
3. 点击 **「测试」**
4. 查看返回结果：

### 验收标准

| 检查项 | 通过条件 |
|--------|---------|
| 返回片段数 | ≥ 1 条（Top 3 中至少 1 条） |
| 最高 Score | ≥ 0.5 |
| 命中文档 | FANUC 0i-MF 维修手册 |
| 片段内容 | 包含「主轴」「报警」「SP-2003」等关键词 |

### 测试查询建议

```
查询 1：主轴异响 报警代码 SP-2003
查询 2：换刀故障 排查步骤
查询 3：驱动过流 IGBT
查询 4：伺服电机过热
查询 5：系统参数备份
```

如果召回测试通过，知识库创建完成，可进入 Workflow 配置阶段（`deploy/部署检查清单.md` 步骤 4）。

## 获取 Knowledge API Key（供检索验证脚本使用）

### 操作指引

1. 在知识库详情页，点击 **「API」** 标签
2. 点击 **「API 密钥」** 旁的 **「创建」** 按钮
3. 复制生成的 API Key（格式：`dataset-xxxxxxxx`）
4. 记录 **Dataset ID**（在 URL 中可以看到，格式：`xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`）

### 用于检索验证脚本

将以下信息配置到环境变量或直接填入 `verify_retrieval.py`：

```bash
DIFY_BASE_URL=http://localhost/v1
DIFY_DATASET_ID=你的dataset_id
DIFY_KNOWLEDGE_API_KEY=dataset-xxxxxxxx
```
