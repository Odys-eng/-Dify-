# 节点配置参数清单

> **文件路径**：`docs/节点配置参数清单.md`
> **用途**：Dify Workflow 6 个节点的所有可配置字段 → 推荐值 → 说明
> **与 PRD 第 3.2 节对照**：所有参数均按 PRD 规定配置

---

## 节点 1：Start（开始）

| 字段 | 推荐值 | 说明 |
|------|--------|------|
| `type` | `start` | 节点类型 |
| `title` | `开始` | 节点标题 |
| `variables` | `[]`（空） | Chatflow 模式下用 `sys.query` 接收用户输入，无需额外变量 |
| `desc` | `接收用户输入的故障描述` | 节点描述 |

**说明**：Chatflow 模式下，用户输入自动存入 `sys.query`，无需在 Start 节点定义输入变量。

---

## 节点 2：Knowledge Retrieval（知识库检索）

| 字段 | 推荐值 | 说明 | PRD 依据 |
|------|--------|------|---------|
| `type` | `knowledge-retrieval` | 节点类型 | — |
| `title` | `知识库检索` | 节点标题 | — |
| `dataset_ids` | （选择实际知识库） | ⚠️ 导入 DSL 后需手动选择 | PRD 4.1 节 |
| `query_variable_selector` | `["sys", "query"]` | 引用用户输入 | — |
| `retrieval_mode` | `multiple` | 多知识库检索模式 | — |
| `multiple_retrieval_config.top_k` | **3** | 返回前 3 个最相关片段 | PRD 3.2 节 |
| `multiple_retrieval_config.score_threshold` | **0.5** | 低于此分不返回 | PRD 3.2 节 |
| `multiple_retrieval_config.reranking_enable` | **true** | 开启 Rerank 提升精度 | PRD 4.3 节 |
| `multiple_retrieval_config.reranking_mode` | `reranking_model` | 使用 Rerank 模型 | — |
| `multiple_retrieval_config.reranking_model` | `bge-reranker-base` | 中文 Rerank 模型 | 知识库调优建议 |

**输出变量**：
- `result`：检索到的片段列表（已按 score_threshold 过滤）

**关键说明**：
> 知识库节点自带 `score_threshold` 过滤，低于 0.5 的结果不会出现在 `result` 中。
> 因此 if-else 节点只需判断 `result` 是否 `not empty`，无需数学比较 score 值。

---

## 节点 3：if-else（条件分支）

| 字段 | 推荐值 | 说明 |
|------|--------|------|
| `type` | `if-else` | 节点类型 |
| `title` | `命中判断` | 节点标题 |
| `cases[0].case_id` | `true` | 分支 ID |
| `cases[0].logical_operator` | `and` | 条件逻辑（单条件无所谓）|
| `cases[0].conditions[0].variable_selector` | `["knowledge_retrieval", "result"]` | 判断知识库输出 |
| `cases[0].conditions[0].comparison_operator` | **`not empty`** | 判断结果是否非空 |
| `cases[0].conditions[0].value` | `""` | not empty 不需要值 |

**分支逻辑**：

```
result not empty（命中，score ≥ 0.5）
  → 直接连接 LLM 节点（使用知识库结果回答）

result empty（未命中，score < 0.5）
  → 连接 HTTP 请求节点（触发联网搜索）
```

**⚠️ 条件表达式语法说明**：

Dify if-else 节点支持的比较运算符：

| 运算符 | 适用类型 | 说明 |
|--------|---------|------|
| `contains` | 字符串 | 包含某值 |
| `not contains` | 字符串 | 不包含某值 |
| `start with` | 字符串 | 以某值开头 |
| `end with` | 字符串 | 以某值结尾 |
| `is` | 字符串/数字 | 等于 |
| `is not` | 字符串/数字 | 不等于 |
| `empty` | 任意 | 为空 |
| `not empty` | 任意 | 不为空 |
| `≥` / `>` / `<` / `≤` | 数字 | 数值比较 |

**本项目使用 `not empty`** 的理由：
1. 知识库节点已按 `score_threshold=0.5` 过滤，`result` 为空意味着无 ≥ 0.5 的命中
2. 避免在 DSL 中写复杂的数学比较表达式
3. 逻辑清晰：有结果 = 命中，无结果 = 未命中

## 节点 4：HTTP Request（Tavily 搜索）

| 字段 | 推荐值 | 说明 | PRD 依据 |
|------|--------|------|---------|
| `type` | `http-request` | 节点类型 | — |
| `title` | `Tavily HTTP 搜索` | 节点标题 | — |
| `method` | `post` | HTTP 方法 | — |
| `url` | `https://api.tavily.com/search` | Tavily API 端点 | PRD 5.1 节 |
| `headers` | `Content-Type: application/json` | 请求头 | — |
| `authorization.type` | `no-auth` | Tavily 用 body 传 Key | — |
| `body.type` | `json` | JSON 格式 | — |
| `body.data.api_key` | **实际 Tavily Key** | ⚠️ 从 .env 复制 | — |
| `body.data.query` | `{{#sys.query#}}` | 搜索查询（引用用户输入）| — |
| `body.data.max_results` | `5` | 返回 5 条结果 | PRD 5.1 节 |
| `body.data.search_depth` | `basic` | 基础搜索（省额度）| — |
| `timeout.max_read_timeout` | `30` | 读取超时 | — |

**输出变量**：
- `body`：HTTP 响应体（Tavily 搜索结果 JSON）
- `status_code`：HTTP 状态码

**⚠️ 说明**：
- Dify 1.16.0 已移除「自定义工具」，改用 `http-request` 节点
- 此节点仅在 if-else `false` 分支（知识库未命中）时执行
- 无需 marketplace 插件，无需 OpenAPI Schema

## 节点 5：LLM（智能问答生成）

| 字段 | 推荐值 | 说明 | PRD 依据 |
|------|--------|------|---------|
| `type` | `llm` | 节点类型 | — |
| `title` | `智能问答生成` | 节点标题 | — |
| `model.provider` | `langgenius/deepseek/deepseek` | Dify 1.16.0 预装的 DeepSeek 原生插件 | — |
| `model.name` | **`deepseek-v4-flash`** | ⚠️ 不用 `deepseek-chat` | PRD 6.1 节 |
| `model.mode` | `chat` | 聊天模式 | — |
| `model.completion_params.temperature` | **0.3** | 降低随机性，提高稳定性 | PRD 6.1 节 |
| `model.completion_params.max_tokens` | **2048** | 输出上限 | — |
| `model.completion_params.thinking.type` | **`enabled`** | 开启思考模式 | PRD 6.1 节 |
| `model.completion_params.reasoning_effort` | **`high`** | 思考强度 | PRD 6.1 节 |
| `vision.enabled` | `false` | 不启用视觉 | — |
| `structured_output_enabled` | `false` | 不强制结构化输出 | — |

**Prompt Template 配置**：

| 角色 | 内容来源 | 说明 |
|------|---------|------|
| `system` | `System-Prompt最终版.md` | ⚠️ 导入后粘贴完整 System Prompt |
| `user` | 模板（含变量引用） | 引用 `sys.query` + 知识库结果 + 搜索结果 |

**User Prompt 模板**：

```
用户问题：{{#sys.query#}}

--- 知识库检索结果（可能为空）---
{{#knowledge_retrieval.result#}}

--- 联网搜索结果（可能为空）---
{{#tavily_search.body#}}

请基于以上检索结果回答用户问题。
如果知识库结果非空，优先使用知识库内容回答。
如果知识库结果为空，使用联网搜索结果回答。
```

**⚠️ 两个分支汇合到此节点**：
- if-else `true`（命中）→ 直接连接 LLM（知识库结果非空，搜索结果为空）
- if-else `false` → HTTP Request → LLM（知识库结果为空，搜索结果非空）

LLM 根据 prompt 中「可能为空」的提示，自动选择非空的数据源。

## 节点 6：Answer（直接回复）

| 字段 | 推荐值 | 说明 |
|------|--------|------|
| `type` | `answer` | 节点类型 |
| `title` | `回复` | 节点标题 |
| `answer` | `{{#llm_generate.text#}}` | 引用 LLM 输出 |
| `variables` | `[]` | 无额外变量 |

**说明**：Answer 节点将 LLM 输出原样返回给用户。引用块（retriever_resource）由 Dify 自动渲染。

---

## 节点连接关系总结

```
Start ──→ Knowledge Retrieval ──→ if-else
                                    │
                    ┌───────────────┴───────────────┐
                    │ true（命中）                    │ false（未命中）
                    │                               │
                    │                          Tavily Search
                    │                               │
                    └───────────────┬───────────────┘
                                    │
                                    ▼
                                  LLM ──→ Answer
```

| 连接 | 源节点 | 目标节点 | 说明 |
|------|--------|---------|------|
| 1 | Start | Knowledge Retrieval | 用户输入 → 检索 |
| 2 | Knowledge Retrieval | if-else | 检索结果 → 判断 |
| 3 | if-else (true) | LLM | 命中 → 直接生成 |
| 4 | if-else (false) | Tavily Search | 未命中 → 联网搜索 |
| 5 | Tavily Search | LLM | 搜索结果 → 生成 |
| 6 | LLM | Answer | 生成 → 输出 |
