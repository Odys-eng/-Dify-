# 性能压测简案

> **文件路径**：`docs/性能压测简案.md`
> **用途**：对 Dify API 进行简单压测，统计响应时间和成功率
> **测试场景**：单用户、10 次循环

---

## 一、压测方案

### 测试参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 并发数 | 1 | 单用户场景（Demo 定位） |
| 测试轮次 | 10 | 连续发送 10 次请求 |
| 测试查询 | 「CNC 加工中心换刀故障」 | 中等难度，覆盖知识库命中 |
| 响应模式 | blocking | 非流式，便于统计完整响应时间 |
| 超时阈值 | 30 秒 | 超过视为失败 |

### 统计指标

| 指标 | 说明 |
|------|------|
| 平均响应时间 | 10 次请求的平均值 |
| P95 响应时间 | 95% 的请求在此时间内完成 |
| 最小响应时间 | 最快一次的响应时间 |
| 最大响应时间 | 最慢一次的响应时间 |
| 成功率 | HTTP 200 的请求占比 |

## 二、压测工具

### 方式 A：Python 脚本（推荐，无需额外安装）

```python
"""
perf_test.py - 性能压测脚本
使用方式：python src/tests/perf_test.py --api_key app-xxx --base_url http://localhost/v1
"""
import requests
import time
import statistics
import argparse
import os

def perf_test(base_url, api_key, rounds=10):
    url = f"{base_url}/chat-messages"
    headers = {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json"
    }
    query = "CNC 加工中心换刀故障"

    results = []
    print(f"开始压测：{rounds} 轮，查询：{query}\n")

    for i in range(rounds):
        payload = {
            "inputs": {},
            "query": f"{query}",  # 每次新对话
            "response_mode": "blocking",
            "conversation_id": "",
            "user": f"perf-test-{i}"
        }

        start = time.time()
        try:
            resp = requests.post(url, headers=headers, json=payload, timeout=30)
            elapsed = round(time.time() - start, 2)
            success = resp.status_code == 200
            results.append({
                "round": i + 1,
                "status": resp.status_code,
                "time": elapsed,
                "success": success
            })
            mark = "✅" if success else "❌"
            print(f"  轮次 {i+1:2d}: {mark} HTTP {resp.status_code} | {elapsed}s")
        except Exception as e:
            elapsed = round(time.time() - start, 2)
            results.append({
                "round": i + 1,
                "status": 0,
                "time": elapsed,
                "success": False
            })
            print(f"  轮次 {i+1:2d}: ❌ 错误 | {elapsed}s | {str(e)[:50]}")

    # 统计
    times = [r["time"] for r in results]
    success_count = sum(1 for r in results if r["success"])

    print(f"\n{'='*50}")
    print(f"  压测结果汇总")
    print(f"{'='*50}")
    print(f"  总轮次：{len(results)}")
    print(f"  成功数：{success_count}")
    print(f"  成功率：{success_count/len(results)*100:.1f}%")
    print(f"  平均响应时间：{statistics.mean(times):.2f}s")
    if len(times) >= 2:
        print(f"  P95 响应时间：{sorted(times)[int(len(times)*0.95)]:.2f}s")
    print(f"  最小响应时间：{min(times):.2f}s")
    print(f"  最大响应时间：{max(times):.2f}s")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--api_key", default=os.environ.get("DIFY_APP_API_KEY", ""))
    parser.add_argument("--base_url", default="http://localhost/v1")
    parser.add_argument("--rounds", type=int, default=10)
    args = parser.parse_args()
    if not args.api_key:
        print("错误：缺少 DIFY_APP_API_KEY")
        exit(1)
    perf_test(args.base_url, args.api_key, args.rounds)
```

### 方式 B：wrk 命令（需安装 wrk）

```bash
# 安装 wrk（Ubuntu）
sudo apt install wrk

# 创建 lua 脚本 dify_test.lua
wrk.method = "POST"
wrk.headers["Content-Type"] = "application/json"
wrk.headers["Authorization"] = "Bearer YOUR_APP_API_KEY"
wrk.body = '{"inputs":{},"query":"CNC 加工中心换刀故障","response_mode":"blocking","conversation_id":"","user":"perf-test"}'

# 执行压测（1 线程，1 连接，10 次请求）
wrk -t1 -c1 -d10s --latency http://localhost/v1/chat-messages
```

### 方式 C：ab 命令（Apache Bench）

```bash
# 创建请求体文件
echo '{"inputs":{},"query":"CNC 加工中心换刀故障","response_mode":"blocking","conversation_id":"","user":"perf-test"}' > /tmp/dify_body.json

# 执行压测（10 次请求，并发 1）
ab -n 10 -c 1 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_APP_API_KEY" \
  -p /tmp/dify_body.json \
  http://localhost/v1/chat-messages
```

## 四、性能基准参考

### 预期性能（基于项目配置）

| 场景 | 预期响应时间 | 说明 |
|------|------------|------|
| 知识库命中 | 3-8 秒 | 无联网搜索，LLM 推理为主 |
| 联网兜底 | 8-15 秒 | 含 Tavily 搜索 + LLM 推理 |
| 多轮追问（第二轮） | 3-8 秒 | 有上下文，略快于首次 |

### 影响因素

| 因素 | 影响程度 | 说明 |
|------|---------|------|
| reasoning_effort=high | ⭐⭐⭐ | 思考模式增加 2-5 秒 |
| Tavily search_depth | ⭐⭐ | advanced 比 basic 慢 2-3 秒 |
| 知识库数据量 | ⭐ | Demo 量级影响小 |
| 网络延迟 | ⭐⭐⭐ | DeepSeek/Tavily API 响应时间 |
| Docker 资源限制 | ⭐⭐ | 内存/CPU 不足导致慢 |

### 优化建议

| 优化项 | 方法 | 预期效果 |
|--------|------|---------|
| 降低思考强度 | reasoning_effort: high → 默认 | 减少 2-5 秒 |
| 限制输出长度 | max_tokens: 2048 → 1024 | 减少 1-2 秒 |
| 简化搜索 | search_depth: basic（已是） | — |
| 增加缓存 | Dify 内置缓存 | 重复查询秒回 |
