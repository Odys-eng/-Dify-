# 分发前清理清单

把项目（仓库或 `DifiProject.rar`）发给别人 / 上传公开仓库前，务必过一遍本清单，
避免泄露 API Key、密码、私钥。

## 一键扫描 / 清理

```bash
# 1. 先扫描（安全，只报告不修改）
bash deploy/sanitize-for-release.sh

# 2. 确认无误后清理（会先备份为 *.release-bak）
bash deploy/sanitize-for-release.sh --apply
```

> Windows 用户在 **Git Bash** 或 **WSL** 下运行。
> `--apply` 会清空本地 `.env` 密钥并移除明文密码/私钥，本地要恢复运行把 `*.release-bak` 改回原名即可。

## 手动核对清单

| 项目 | 位置 | 处理 |
|------|------|------|
| 大压缩包 | `DifiProject.rar` | 已在 `.gitignore`，不入库；发包时确认包内 `dify/docker/.env` 等已清理 |
| 环境变量密钥 | `dify/docker/.env` | 清空 `DEEPSEEK/TAVILY/SILICONFLOW_API_KEY`、`SECRET_KEY`、`DB/REDIS/WEAVIATE` 等密码 |
| Basic Auth 明文密码 | `dify/docker/nginx/conf.d/.htpasswd_plaintext` | 移除（接收方重新生成） |
| Basic Auth 哈希 | `dify/docker/nginx/conf.d/.htpasswd` | 可保留（是哈希），或一并重置 |
| SSL 私钥 | `dify/docker/nginx/ssl/dify.key` | 移除（接收方重新生成自签证书） |
| Tavily 临时文件 | `key.txt` | 删除（已在 `.gitignore`） |
| IDE 本地配置 | `.claude/settings.local.json` | 含真实 Weaviate Key，已在 `.gitignore`，不入库 |

## 上传公开仓库前额外确认

```bash
# 确认没有敏感文件会被提交（应只看到 .env.template 一类占位符）
git status
git ls-files | grep -iE "\.env$|htpasswd|\.key$|\.rar|settings\.local" || echo "✓ 干净"
```

- 真实密钥一旦推到公开仓库，即使删除也可能被缓存/爬取，务必**推送前**确认。
- 若已误推，需在对应平台**吊销并轮换**该密钥，而不是仅删文件。
